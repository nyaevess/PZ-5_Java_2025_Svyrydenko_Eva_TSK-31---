№1. Порівняйте та назвіть переваги та недоліки таких методів сортування: сортування бульбашкою, сортування вибором, сортування вставкою.

Сортування бульбашкою:
Переваги — простий у реалізації, підходить для невеликих масивів, може завершитися раніше, якщо масив уже частково відсортований.
Недоліки — дуже повільний для великих масивів, виконує багато непотрібних обмінів, має складність O(n²).

Сортування вибором:
Переваги — проста реалізація, виконує мінімальну кількість обмінів.
Недоліки — повільний для великих масивів, завжди має складність O(n²), не є стабільним (однакові елементи можуть змінити порядок).

Сортування вставкою:
Переваги — добре працює для невеликих або частково відсортованих масивів, стабільний, простий у реалізації.
Недоліки — неефективний для великих невпорядкованих масивів, має складність O(n²) у гіршому випадку.

№2. Оцініть кожен з методів відповідно до наступних критеріїв:

час роботи;

потреби у додатковій пам’яті;

стабільність.
За якими ще критеріями ви можете порівняти ці алгоритми?

За часом роботи всі три алгоритми у середньому мають складність O(n²), але сортування вставкою може працювати швидше на частково відсортованих даних — тоді воно має складність O(n).
За потребами у додатковій пам’яті всі три методи використовують лише постійну кількість пам’яті, тобто O(1).
За стабільністю: сортування бульбашкою та вставкою — стабільні, а сортування вибором — нестабільне.
Також алгоритми можна порівняти за кількістю обмінів і порівнянь, простотою реалізації та придатністю до потокової обробки даних.

№3. Який випадок є найкращім або найгіршим для роботи цих алгоритмів? (Наприклад: частково відсортований масив або масив, відсортований у зворотному порядку). Який з методів забезпечить у цих випадках найкращій/найгірший результат?

Для сортування бульбашкою найкращий випадок — вже відсортований масив, бо алгоритм може завершитися раніше. Найгірший випадок — коли елементи розташовані у зворотному порядку.
Для сортування вибором найкращий і найгірший випадок однакові, бо кількість порівнянь завжди фіксована незалежно від упорядкованості.
Для сортування вставкою найкращий випадок — частково відсортований масив, тоді алгоритм працює швидко. Найгірший випадок — масив, відсортований у зворотному порядку.

---
№4. Що означають позначення O(1), O(n), O(n²)?

Ці позначення описують складність алгоритму за часом виконання.
O(1) означає, що час виконання не залежить від кількості елементів.
O(n) означає, що час зростає прямо пропорційно кількості елементів у масиві.
O(n²) означає, що час зростає пропорційно квадрату кількості елементів — при подвоєнні розміру масиву час виконання зростає приблизно у чотири рази.
